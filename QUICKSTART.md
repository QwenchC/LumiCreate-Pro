# LumiCreate-Pro 30 分钟上手 · 从 0 到第一个漫剧

本文带你跑通：**装好 → 配引擎 → 生成一个 1 分钟漫剧**。完整功能见 [README.md](README.md)。

> 默认目标：**纯朗读漫剧**（最容易跑成功的形态：文案直读、按句切镜、首末帧锁角色一致性）

---

## 0. 你需要准备的

| 组件 | 用途 | 必需？ |
|------|------|--------|
| **Windows 10/11** | 桌面端 | 是 |
| **ComfyUI**（已带 LumiCreate 工作流） | 图片 + 视频生成 | 是 |
| **文本引擎账号**（DeepSeek 推荐） | 写文案、出英文 prompt、检测角色 | 是 |
| **微软神经语音**（edge-tts 内置，无需账号） | 朗读音频 | 是（默认）|
| IndexTTS-2.0 / GPT-SoVITS（本地） | 替代 edge-tts（需要克隆音色时） | 否 |
| 5 GB 空闲磁盘 | 项目 + 模型缓存 | 是 |

## 1. 启动 LumiCreate

```powershell
# 仓库根目录
npm install                    # 第一次需要
npm run dev                    # 同时启 backend + electron renderer
```

启动后 Electron 窗口 + backend 在 `http://127.0.0.1:18520` 上跑。

---

## 2. 一次性配置（设置页）

打开 ⚙ 设置。

### 2.1 文本引擎
- **引擎类型**：DeepSeek
- **Base URL**：`https://api.deepseek.com`
- **API Key**：自己的 key
- **模型**：`deepseek-chat`
- **批量并发数**：**100**（v4-flash 上限是 2500，100 已经让全片 30 镜的 prompt 生成 < 10s）
- 点 **🔌 测试** 看到 "✓ 连接成功"

### 2.2 图片引擎（ComfyUI）
- **ComfyUI 地址**：`http://localhost:8188`（默认）
- **工作流文件夹**：`<你的 ComfyUI>/user/default/workflows`
- **默认工作流**：`lumicreate-基础`
- **图片尺寸**：默认 1920×1080（漫剧后续会按竖屏自动切）

先确保 ComfyUI 启动并能在浏览器打开 `http://localhost:8188`。

### 2.3 语音引擎
- **引擎类型**：**微软神经语音（Edge TTS）**
- 点 **🧪 测试全部 14 个音色** —— 大约 10 秒，把当前不可用的音色筛掉
- **默认音色**：晓晓（自然女声，漫剧叙述常用）
- **默认语速**：**+25%（快，漫剧默认）**
- 点 **▶ 试听** 听一段，确认网络通

### 2.4 视频引擎（ComfyUI）
- **ComfyUI 地址**：同上
- **工作流文件夹**：留空（自动从图片引擎继承）
- **ComfyUI input/**：填 `<你的 ComfyUI>/input`（**必填**，LumiCreate 的视频音频通过写入 ComfyUI input 上传）
- **默认工作流**：`flfa2i-lumicreate`
- **分辨率**：**720×1280（竖屏 HD）** ← 漫剧默认
- **FPS**：25

## 3. 新建项目

回到主屏，点 **+ 新建项目**。

- **项目名称**：随便，"测试漫剧"
- **对白模式**：**纯朗读（直读文案）** ← **关键**：漫剧首选 reading 模式

### 3.1 文案
两种方式：
1. **直接粘贴**已有文案到「文案配置」tab 的输入框
2. **AI 生成**：填几行设定，点"基于配置生成"

300-500 字够 1 分钟漫剧。

### 3.2 角色（决定图片 + 音色一致性）
进「角色管理」tab。如果文案里有角色名：
- 点 **+ 添加角色** → 填名字、定位、性格
- 点 **🎨 生成 appearance**（英文视觉标签，会注入图片 prompt）→ 出图时锁定该角色样貌
- **微软神经语音**字段：男角色配云希/云健，女角色配晓晓/晓涵，童声配云夏
  - 没角色名（纯叙述）→ 跳过这步

> ⚠ appearance 字段**必须非空**才会让图片角色稳定，建议每个角色都点一次"生成 appearance"。

### 3.3 分镜
进「分镜设计」tab。
- 点 **✂ 从文案手动生成分镜** —— 按标点切镜，每镜约 12 秒，更安全（自动生成会让 LLM 拼长镜，单镜超过 LTX 上限）
- 点 **🔍 自动检测角色** —— LLM 给每镜填 `_scene_characters`（识别"他/她"指代）
- 点 **🖼 生成全部提示词** —— 批量调 LLM 生成 start/end frame 的英文 prompt，hydrate 角色 appearance

跑完每镜会显示首/末帧 prompt，可手动微调。

### 3.4 图片
进「图片生成」tab。
- **工作流**：`lumicreate-基础`
- 点 **▶ 批量生成** —— ComfyUI 一镜出一张首帧 + 一张末帧
- 每张图右上角点 **选中** 决定用哪张（每镜可生成多个版本对比）

### 3.5 音频
进「音频生成」tab。
- 顶部显示 "📖 朗读模式 · 微软神经语音 · N 段"
- **叙述音色**：晓晓 / **语速**：快 / **区分音色**：打开
- 出现"对话音色"下拉 + 下面的"👥 角色音色矩阵"
- 角色音色矩阵：检查每个角色的 voice 是否合适，点 **▶ 试听** 听单个角色
- 点 **▶ 全部生成语音** —— 几秒到几十秒

每镜一行，<audio> 播放器直接听。

### 3.6 视频
进「视频生成」tab。
- **工作流**：`flfa2i-lumicreate`
- **分辨率**：720×1280
- **FPS**：25
- 点 **▶ 批量生成视频** —— 每镜调 ComfyUI 走 LTX 工作流，首末帧 + 音频 → 一段 mp4。GPU 上每镜约 30-90s

### 3.7 字幕
进「字幕生成」tab。
- **字号**：竖屏漫剧用 **10**，横屏用 16（重要——18 会盖住竖屏画面）
- **字体**：等线 Bold
- 点 **断句工具** → 📋 从文案复制 → 应用 ← 用 `,。？！：""——…` 把文案切成短行字幕
- **FPS** = 25（必须和视频一致）
- 点 **▶ 生成 SRT** → SSE 走 ffmpeg + stable-whisper 生成 SRT
- 点 **▶ 烧入视频** → ffmpeg 把字幕烧到 mp4

### 3.8 合并最终视频
进「视频生成」tab 底部，点 **合并所有镜** —— 把每镜带字幕的视频 concat 成成片。

## 4. 失败镜的"局部重试"

任何一个批量阶段（图片 / 音频 / 视频）跑完，如果有失败镜：
- 顶部会显示"上次失败：N 镜 [↻ 重试失败镜]"
- 点重试 —— 只对失败的 scene_id 重新生成，已成功的不会重跑
- 后端持久化在 `<项目>/last_run_errors.json`，刷新页面或重启都还在

## 5. 提速 / 排错速查

| 现象 | 检查 |
|------|------|
| 文本生成卡 / 慢 | 设置文本引擎并发数 → 100+ |
| ComfyUI 出图返回 random 内容 | 分镜的 `start_frame_prompt` 是否非空 → 点"生成全部提示词" |
| 角色样貌每镜都变 | 角色 appearance 字段是否填了；分镜 `_scene_characters` 是否选了 |
| msedge 试听一些声音报 503 | 设置页 → "测试全部音色" 重跑，淘汰失败的 |
| 视频生成全部失败 | ComfyUI input 目录是否填对 + ComfyUI 是否运行；试着重启 ComfyUI |
| LTX 视频报"应当相同" | VRAM 不够；LumiCreate 已经自动 `/free` 重试一次，仍失败说明该镜 prompt 太复杂 |
| audio.json 几百 MB | reading 模式的 mp3 base64；轮 2 之后会拆文件落盘 |
| 顶部窗口拖不动 | 已修；按住 logo 区域 / 标签右侧空白都能拖 |

# LumiCreate-Local 项目开发文档

## 1. 项目概述

### 1.1 项目简介
LumiCreate-Local 是 LumiCreate 开源项目的本地版本，是一个面向漫画剧集生成的桌面应用软件。相比于云端版本，本地版提供离线工作能力，整合多个专业的内容生成引擎，支持用户从文案创意到最终视频的全流程本地生成。

**项目名称**：LumiCreate-Local  
**开源仓库**：https://github.com/QwenchC/LumiCreate  
**目标平台**：Windows 11  
**发布方式**：安装包（.exe / .msi）  

### 1.2 核心功能
- **文案创建**：支持手动编写或 LLM 辅助生成
- **分镜设计**：LLM 根据文案自动设计分镜结构
- **图片生成**：基于分镜提示词批量生成图像
- **音频生成**：角色语音合成（基于角色和情绪）
- **视频生成**：将所有素材合成最终视频

### 1.3 用户场景
- **内容创作者**：快速将创意转化为视频作品
- **剧本作者**：创作并可视化自己的故事
- **个独UP主**：降低漫剧制作成本，实现自主创作

### 2.3 工作流程

```
用户创建项目
    ↓
[文案创建] → LLM 生成文案 → 用户编辑确认
    ↓
[分镜设计] → LLM 生成分镜+台词 → 用户确认
    ↓
[图片生成] → 批量生成 → 用户选图（首/尾帧）
    ↓
[音频生成] → 角色语音合成 → 用户选择最佳版本
    ↓
[视频生成] → 合成最终视频
    ↓
项目保存
```

#### 工作流程（关键）
1. 用户点击 [批量生成]
2. 系统遍历每个分镜的首尾帧
3. **并行生成**：每张图片同时创建 N 个生成任务（N=生成次数）
4. 实时显示生成进度和缩略图
5. 用户可随时中断、继续或单独重新生成某张图片
6. 最后为每个分镜的首尾帧各选用一张最终图片

## 6. 外部引擎集成

### 6.1 文本生成引擎

#### Ollama 集成
- **连接方式**：HTTP API
- **端点**：`http://localhost:11434/api/generate`
- **支持离线运行**

#### LM Studio 集成
- **连接方式**：HTTP API
- **端点**：`http://localhost:1234/v1/chat/completions`
- **OpenAI 兼容格式**

#### DeepSeek API 集成
- **连接方式**：HTTPS API
- **端点**：`https://api.deepseek.com/v1/chat/completions`
- **需要 API Key**

### 6.2 图片生成引擎 - ComfyUI

#### 连接方式
- **本地 WebSocket 连接**
- **端点**：`ws://localhost:8188`
- **HTTP API**：`http://localhost:8188/api`

#### 关键 API
- 获取工作流列表：`GET /api/workflows`
- 获取模型列表：`GET /api/models`
- 提交任务：`POST /api/prompt`
- 查询进度：`WebSocket` 消息

### 6.3 语音生成引擎 - GPT-SoVITS

#### 连接方式
- **HTTP API（推荐）**
- **端点**：`http://localhost:9880/api`

#### 关键 API
- 获取模型列表：`GET /api/models`
- 获取参考音频：`GET /api/reference_audios`
- 生成语音：`POST /api/tts` 
  ```json
  {
    "model_id": "model_v1",
    "text": "早上好呀~",
    "reference_audio": "joy_01.wav",
    "emotion_params": {...}
  }
  ```

### 6.4 视频生成引擎 - ComfyUI

#### 工作流设计（示例）
- 输入：图片序列、音频、文案
- 处理：图片帧处理、音视频同步
- 输出：MP4 视频文件

## 7. 工作流详解

### 7.1 完整项目创建流程

```
1. 新建项目
   └─> 输入项目名称、描述
   
2. 文案创建
   ├─> 选项A：手动输入文案
   └─> 选项B：LLM 辅助生成
       ├─> 配置文案参数（篇幅、受众、风格）
       ├─> LLM 生成初稿
       └─> 用户编辑和微调
   
3. 分镜设计
   ├─> LLM 根据文案自动分镜
   ├─> 生成每个分镜的：
   │   ├─ 场景描述
   │   ├─ 首尾帧提示词
   │   └─ 角色台词（含情绪）
   └─> 用户编辑和调整
   
4. 图片生成
   ├─> 选择 ComfyUI 工作流
   ├─> 设置生成次数（默认3）
   ├─> 批量生成所有分镜的首尾帧
   │   └─> 每张图片同时生成N次
   ├─> 实时显示进度和缩略图
   └─> 用户为每个分镜选用最佳首尾帧图片
   
5. 音频生成
   ├─> 为每个台词配置：
   │   ├─ 声音模型
   │   └─ 情绪参考音频
   ├─> 批量生成语音（默认3版本）
   └─> 用户选择最佳版本
   
6. 视频生成
   ├─> 配置视频参数
   ├─> ComfyUI 合成所有素材
   └─> 生成最终视频
   
7. 保存&导出
   └─> 保存项目或导出视频文件
```

### 7.2 编辑时保存提醒

- **检测方式**：监听用户的所有编辑操作
- **未保存标记**：在标题栏显示 `*` 或 `(未保存)`
- **退出提醒**：
  ```
  检测到项目有未保存的更改，是否保存？
  [保存] [不保存] [取消]
  ```

## 8. 功能特性与创新点

### 8.1 核心创新机制

#### 并行图片生成
- **传统方式**：生成一张图片后再生成下一张（串行）
- **LumiCreate 方式**：**每张图片同时生成多次，所有分镜同时进行**
  ```
  并行度示意：
  时间线 →
  第1分钟: [S1首#1] [S1首#2] [S1首#3] [S1尾#1] [S2首#1] ...
  第2分钟: [S1尾#2] [S1尾#3] [S2首#2] [S2首#3] ...
  ```
- **优势**：
  - 充分利用 GPU 资源
  - 大幅减少总生成时间
  - 给用户更多选择

#### 实时进度反馈
- 实时显示生成中的缩略图
- 进度条分层展示（总体 + 单分镜）
- 支持随时中断和继续

#### 灵活的多引擎架构
- 支持多个文本生成源（离线 + 在线）
- 未来可扩展其他图片生成器
- 模块化设计便于维护和升级

## 9. 技术要点与挑战

### 9.1 技术挑战

| 挑战 | 解决方案 |
|------|---------|
| **并行任务管理** | 使用队列系统 (Celery/APScheduler) 管理生成任务 |
| **实时进度同步** | WebSocket 或 Server-Sent Events (SSE) 实时推送 |
| **大文件处理** | 分片上传下载，断点续传机制 |
| **GPU 资源竞争** | 任务优先级队列，防止资源耗尽 |
| **多引擎切换** | 适配器模式，统一接口层 |
| **项目文件管理** | 原子操作，事务日志保障数据一致性 |

### 9.2 性能优化

1. **缓存策略**
   - 缓存 LLM 的生成结果
   - 缓存模型列表和参考资源

2. **异步处理**
   - 所有 I/O 操作异步化
   - 不阻塞 UI 线程

3. **数据库优化**
   - 本地 SQLite，简化部署
   - 适当索引加速查询

## 13. 常见问题 (FAQ)

**Q: 为什么选择本地部署而不是云端？**
A: 本地部署优处：隐私、无网络延迟、降低用户成本、离线工作能力。

**Q: 图片生成为什么要生成多次？**
A: 用户可获得更多选择，挑选最满意的结果，提高最终作品质量。

**Q: 每个分镜的图片数量是多少？**
A: 默认每分镜生成 2 张图片（首帧+尾帧）。每张图片默认生成 3 个版本供选择。

**Q: 支持哪些音频模型？**
A: 目前主要基于 GPT-SoVITS，用户可导入或通过 API 调用。

**Q: 视频生成速度有多快？**
A: 取决于项目复杂度、GPU 性能和 ComfyUI 工作流配置。通常 5 分钟视频需 30-60 分钟。

## 附录：关键术语

| 术语 | 定义 |
|------|------|
| **分镜** | 故事的一个片段，包含场景、角色、台词等 |
| **提示词** | 给图片生成引擎的描述文本 |
| **工作流** | ComfyUI 中的节点组合，定义图片/视频生成过程 |
| **声音模型** | GPT-SoVITS 中预训练的语音克隆模型 |
| **情绪标注** | 角色台词的情感状态（喜、怒、哀、乐等） |
| **版本** | 同一内容的多个生成结果 |
